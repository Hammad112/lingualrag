# Deploying LingualRAG to Render free tier

Total time: ~20 minutes. Total cost: **$0/month**.

Architecture in production:

```
Browser ── Vercel (Next.js) ──HTTPS──▶ Render (FastAPI, 512MB) ──▶ Render Postgres (free 1GB)
                                              │
                                              ├──▶ HuggingFace Inference API (embeddings, free)
                                              ├──▶ Qdrant Cloud (vectors, 1GB free)
                                              └──▶ Groq (LLM, free)
```

Why this split? Embeddings run on the **HuggingFace Inference API** (free, no
local model) so the backend's RAM footprint is ~220 MB — comfortably inside
Render's 512 MB cap.

---

## 1 — Sign up for the free services (10 min)

| Service | URL | What you copy |
|---|---|---|
| HuggingFace | https://huggingface.co/settings/tokens | A "read" access token |
| Qdrant Cloud | https://cloud.qdrant.io | Cluster URL + API key (free 1GB cluster) |
| Groq | https://console.groq.com/keys | API key |
| Resend (optional, real emails) | https://resend.com/api-keys | API key |
| Render | https://dashboard.render.com | Connect your GitHub |

Tips while signing up:
- **Qdrant**: pick the region nearest to Render Oregon (us-east or us-west). Wait
  ~2 min for the cluster to boot, then create the API key.
- **Resend**: skip for now — the backend prints OTPs to its log so you can
  copy them from Render's "Logs" tab during development.

## 3 — Deploy via Render Blueprint (one click)

1. Render dashboard → **New +** → **Blueprint**.
2. Pick your GitHub repo. Render reads [`Backend/render.yaml`](render.yaml) and
   proposes:
   - A web service `lingualrag-backend`
   - A free Postgres `lingualrag-db`
3. Click **Apply**. Render starts both, but the build will pause asking you to
   fill in the `sync: false` env vars in the dashboard:

| Variable | Where it comes from |
|---|---|
| `HF_API_TOKEN` | HuggingFace access token from step 1 |
| `QDRANT_URL` | Qdrant cluster URL (looks like `https://xxx.cloud.qdrant.io:6333`) |
| `QDRANT_API_KEY` | Qdrant API key |
| `GROQ_API_KEY` | Groq API key |
| `RESEND_API_KEY` | Resend key (optional — leave blank for console OTP) |

Click **Save Changes**. The service redeploys (~3 min) and you'll get a URL like
`https://lingualrag-backend.onrender.com`.

The `DATABASE_URL`, `SECRET_KEY`, and `JWT_SECRET` are wired/generated by Render
automatically — you don't touch them.

---

## 4 — Tables are created on first boot

Render starts the service → the lifespan hook in [`app/main.py`](app/main.py) runs
`init_db()`, which `CREATE TABLE`s every model against your Postgres. Watch the
**Logs** tab for `Application startup complete` to confirm.

---

## 5 — Deploy the frontend (Vercel — 3 min)

1. Vercel dashboard → **Add New… → Project** → import the same GitHub repo.
2. **Root directory**: `Frontend` (important — the repo has both).
3. Framework: Next.js (auto-detected).
4. **Environment Variables**:
   - `NEXT_PUBLIC_API_URL` = `https://lingualrag-backend.onrender.com`
   - `NEXT_PUBLIC_APP_NAME` = `LingualRAG`
5. Click **Deploy**. You get `https://<project>.vercel.app`.

Now update the backend's CORS list to your Vercel domain:

- Render → `lingualrag-backend` → **Environment** → edit `BACKEND_CORS_ORIGINS`:
  ```json
  ["https://<project>.vercel.app","http://localhost:3000","http://localhost:3001"]
  ```
- Click **Save Changes** (triggers a redeploy).

---

## 6 — Stop the free tier from sleeping

Render free web services sleep after 15 min of no traffic, then take ~30 s to
wake up. The fix is a no-op cron ping:

1. https://cron-job.org → free account.
2. **Create cron job** → URL `https://lingualrag-backend.onrender.com/health`.
3. Schedule: **every 10 minutes**.
4. Save.

The endpoint returns a tiny JSON, costs nothing, and keeps the dyno warm.

## 7 — Promote your first admin user

1. Sign up on the live site with your email.
2. Render dashboard → `lingualrag-db` → **Shell** (or use any Postgres GUI):
   ```sql
   UPDATE users SET is_admin = true WHERE email = 'you@example.com';
   ```
3. Refresh — `/admin` is now visible to you.

## 8 — Things to know about the free tier

- **Cold start**: 30 s the first time after sleep. The cron-job.org pinger
  eliminates this for normal users.
- **Postgres lifespan**: Render's free Postgres **expires after 90 days**. Before
  it expires, run `pg_dump` from the **Shell** and restore into a fresh free
  Postgres. Or upgrade to $7/month for permanent.
- **Uploads are ephemeral**: `PERSIST_UPLOADS=false` is set in `render.yaml`.
  Files are processed in-memory and discarded — only the chunks + vectors land
  in Postgres/Qdrant, which is all you need for retrieval.
- **HuggingFace cold start**: the first embedding call after a quiet period
  returns 503 while the model warms; the client retries automatically.

## 9 — Local dev

```bash
cd Backend
pip install -r requirements.txt
cp .env.example .env
# add HF_API_TOKEN to .env (free at huggingface.co/settings/tokens)
uvicorn app.main:app --reload
```

Local dev uses **SQLite + console OTP + the HuggingFace Inference API for
embeddings**, so the only cloud dependency you need to develop is a free
HuggingFace token (30 seconds to create). Postgres, Qdrant Cloud, Groq, and
Resend stay disabled by default.

## 10 — Troubleshooting

| Symptom | Fix |
|---|---|
| `OOM killed` in Render logs | The new prod requirements no longer contain torch / sentence-transformers; if you see this, something else (maybe an oversized PDF) is the cause — check the Logs tab around the time of the OOM. |
| `HF API error 503` then never recovers | The model needs to warm; first call after a long idle can take 30 s. Subsequent calls are fast. If it persists, your `HF_API_TOKEN` is wrong. |
| `CORS error` in browser | Add the exact Vercel URL to `BACKEND_CORS_ORIGINS` and redeploy. |
| Frontend can't reach backend | Confirm `NEXT_PUBLIC_API_URL` on Vercel has **no trailing slash** and uses **https**. |
| Database tables missing | First boot creates them via `init_db()`. If the logs show an error, check the `DATABASE_URL` env var was wired correctly. |
